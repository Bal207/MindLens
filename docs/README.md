# MindLens — Landing Page (built with Doxygen)

The Apple-style landing page with the download button is generated by
**Doxygen**. Doxygen takes the content in `mainpage.dox`, wraps it with the
custom `header.html` / `footer.html`, and applies `landing.css` + `landing.js`
to produce a standalone site in `html/`.

## Build

```bash
cd docs
doxygen Doxyfile
```

Output is written to `docs/html/`. Open `docs/html/index.html` (or host the
whole `html/` folder on any static host / GitHub Pages).

```bash
# preview locally
cd docs/html && python3 -m http.server
```

## Set the download target (one line)

The download button auto-detects the visitor's OS and links to your **GitHub
Releases**. Set your repo once in [`landing.js`](landing.js):

```js
const GITHUB_REPO = "YOUR_GITHUB_USERNAME/MindLens";
```

Then publish a GitHub Release with assets named `MindLens-macOS.dmg`,
`MindLens-Windows.zip`, `MindLens-Linux.tar.gz` (these names are configurable
in the `ASSETS` map in `landing.js`). Re-run `doxygen Doxyfile` after editing.

## Files

| File           | Purpose                                                        |
|----------------|----------------------------------------------------------------|
| `Doxyfile`     | Doxygen config (HTML-only, chrome disabled, custom templates). |
| `mainpage.dox` | The landing markup, passed through verbatim via `\htmlonly`.   |
| `header.html`  | Minimal `<head>` + `<body>` open (no Doxygen nav/search).      |
| `footer.html`  | Loads `landing.js`, closes the document.                       |
| `landing.css`  | Apple-style styles + scroll animations + Doxygen chrome resets.|
| `landing.js`   | OS detection, GitHub Releases links, scroll-reveal/animations. |
| `html/`        | **Generated** output — the site to deploy.                     |
